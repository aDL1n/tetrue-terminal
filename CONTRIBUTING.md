# Contributing

## Development Branches

- `main` contains stable releases
- `dev` contains active development

## Making Changes

1. Fork the repository
2. Create a feature branch from `dev`
3. Make your changes
4. Open a pull request targeting `dev`

## Reporting Issues

Please open an issue before working on large changes

This helps ensure that proposed changes align with the project's direction

## Coding Guidelines

- Follow existing code style where possible
- Keep changes focused and relevant to the PR
